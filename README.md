# Loubfy OS

🇧🇷 [Versão em português](README.pt-BR.md)

Loubfy OS is a reusable content operating system for social media and blog production with AI agents.

It stores brand context, audience research, editorial rules, visual references, content drafts, renderable assets, workflow docs, and reusable skills for any business, creator, page, or client configured through onboarding.

Loubfy OS is the name of the reusable project/product — onboarding sets the actual business or social profile being managed.

## Who This Is For

- Creators and teams building a repeatable content workflow.
- Agencies that want one operating model per client or brand.
- AI-agent users who want structured context, reusable skills, and predictable file boundaries.
- Developers who want renderable social assets powered by versioned content files.

## Requirements

- Git
- Node.js 20+
- npm
- An AI coding/agent interface such as Claude Code, Codex, Gemini CLI, OpenCode, or another agent that can read repository instructions.

## Quick Start

```bash
git clone https://github.com/victorbenazzi/loubfy-os my-content-os
cd my-content-os
npm install
npm run typecheck
```

`npm install` also links the native skill layer for Claude Code (see [How Agents Use This Repo](#how-agents-use-this-repo)).

Then ask your agent:

```text
Read AGENTS.md and run onboarding to configure this project.
```

If you are using Claude Code, ask it to read `CLAUDE.md`. If you are using Gemini-style agents, ask them to read `GEMINI.md`.

To run Loubfy OS for more than one brand or client, clone it once per brand into a separate root folder — do not mix two businesses in one repository.

## How Agents Use This Repo

Loubfy OS is **agent-agnostic** and **skill-first**. It works the same way across Claude Code, Codex, Gemini CLI, OpenCode, Hermes, and other agent interfaces.

### Authoritative sources

These files are the single source of truth and work in any agent — no install required, an agent just reads them:

- `AGENTS.md` — project rules, structure, and operating model.
- `skills/{skill-name}/SKILL.md` — the durable process for each capability.
- `agents/roster/` — portable role definitions for delegated work.

### Native agent layers

Some interfaces support native skills, subagents, slash commands, and hooks. This repo ships per-agent layers that are **purely additive** — an agent that does not support a layer simply ignores it. The layers only point back to the authoritative sources; they never fork them.

**Claude Code** — the `.claude/` layer is ready after `npm install`, on macOS, Linux, and Windows:

| Path | What it gives you |
|---|---|
| `.claude/skills/` | A link to `skills/` so skills trigger natively. Generated by `npm install` (symlink on macOS/Linux, junction on Windows) — not committed. Rebuild anytime with `npm run setup:claude`. |
| `.claude/agents/` | The `agents/roster/` roles as real subagents for parallel, independent subtasks. |
| `.claude/commands/onboarding.md` | Registers the `/onboarding` slash command. |
| `.claude/settings.json` | A permission allowlist for the project's npm scripts, plus a `PreToolUse` hook that asks for confirmation before any edit inside `references/templates/` (protected source-of-truth templates). |

Never edit files under `.claude/skills/` — edit `skills/` instead.

**Other agents** — install the skills natively with:

```bash
npx skills add . --agent <codex|gemini|...> --copy
```

### Operating model

- Use the smallest execution model that produces a high-quality result: one skill for a narrow task, chained skills for multi-step work, subagents only for independent parallel workstreams.
- The main agent acts as editor-in-chief: it routes work, consolidates outputs, protects file boundaries, and runs the final quality pass.
- No external swarm runtime is required.

See `workflows/agent-orchestration.md` for the full orchestration protocol.

## Initialization

Before strategic content work, agents check:

```text
memory/project-state.yml
```

If `loubfy_os_initialized` is not `true`, run the onboarding skill (or `/onboarding` in Claude Code) before producing strategic content. Onboarding interviews the user and populates `context/`, `brand/`, `guidelines/`, `hooks/`, and `memory/`.

## Repository Model

```text
context/          Company, audience, offer, channel, and page context.
brand/            Design system, branding, colors, typography, logos, visual rules.
guidelines/       Voice, tone, content rules, and guardrails.
images/           Generated, sourced, working, and exported image assets.
references/       External inspiration, screenshots, source notes, and templates.
content/          Drafts and final content for social media, blogs, and newsletters.
memory/           Persistent decisions, state, variables, and recurring context.
agents/           Portable agent cards, handoff templates, and role definitions.
skills/           Reusable agent skills, each with its own SKILL.md.
workflows/        Operating models and workflow documentation.
scripts/          API calls, automations, and repeatable utilities.
hooks/            Social hook formulas, tested hooks, and hook patterns.
.claude/          Native Claude Code layer (additive — other agents ignore it).
```

## Skills

Each skill is a self-contained capability in `skills/{skill-name}/SKILL.md`:

| Skill | Use it to |
|---|---|
| `onboarding` | Configure or reconfigure the project for a business, creator, or client. |
| `audience-research` | Map ICP, pains, objections, JTBD, and voice-of-customer. |
| `content-strategy` | Define pillars, topic clusters, calendars, and backlog. |
| `creative-angles` | Generate hooks, angle matrices, and test hypotheses. |
| `social-copywriting` | Write platform-native posts, captions, hooks, CTAs, and image text. |
| `carousel-builder` | Build slide-by-slide carousel structure and text. |
| `visual-briefing` | Plan image direction, prompts, and visual specs. |
| `social-distribution` | Repurpose one asset across platforms and plan publishing. |
| `collab-campaigns` | Plan co-marketing and creator collaborations. |
| `performance-review` | Interpret metrics and decide what to repeat or test. |

List discoverable skills:

```bash
npm run skills:list
```

## Content Paths

Social content:

```text
content/social-media/{platform}/{accountId}/{slug}/
```

```text

copy.md
carousel.md
image-brief.md
```

Blogposts:

```text
content/blogposts/{slug}/
```

Recommended files:

```text

outline.md
draft.md
```

Use kebab-case slugs (`comparativo-ferramentas-ia`). Always use an explicit `accountId`, even when a platform has only one profile.

## Quick Account Setup

Create a managed social account folder and its context file:

```bash
npm run account:create -- \
  --platform linkedin \
  --account-id principal \
  --display-name "Main Profile" \
  --notes "Brand's main account"
```

This creates:

```text
content/social-media/linkedin/principal/
context/accounts/linkedin-principal.md
```

## Rendering

This template repository does not ship posts under `content/`. Create a post in a new `content/social-media/{platform}/{accountId}/{slug}/` folder, then:

```bash
# Validate a post file
npm run validate:post -- --input content/social-media/linkedin/principal/example/post.json

# Render an image
npm run render:image -- --input content/social-media/linkedin/principal/example/post.json

# Render a video
npm run render:video -- --input content/social-media/linkedin/principal/example/post.json
```

## Agent Instruction Files

| File | Read by |
|---|---|
| `AGENTS.md` | Codex, OpenCode, Hermes, and generic agents — the source of truth. |
| `CLAUDE.md` | Claude Code — points back to `AGENTS.md` and documents the `.claude/` layer. |
| `GEMINI.md` | Gemini CLI and Gemini-style agents — points back to `AGENTS.md`. |

When instructions conflict, follow this order: current user request → `AGENTS.md` → relevant skill → folder-specific context files.

## Development

```bash
npm install        # installs dependencies and links the .claude/ skill layer
npm run typecheck  # type-checks the post/render pipeline
```

Useful scripts:

```bash
npm run account:create   # create a managed social account
npm run validate:post    # validate a post.json file
npm run render:image     # render a post to an image
npm run render:video     # render a post to a video
npm run skills:list      # list discoverable skills
npm run setup:claude     # rebuild the .claude/skills link (run if skills are missing)
```

## Making This Safe To Publish

Before sharing the repository publicly or turning it into a template:

- Remove private client/company data from `context/`, `brand/`, and `memory/`.
- Keep `content/` empty except for `.gitkeep` files; canonical visual templates live in `references/templates/`.
- Remove secrets, API keys, cookies, and local environment files.
- Keep generated final assets only when they are part of the public demo.
- Choose and add a license if other people are allowed to use, modify, or redistribute the project.
- Confirm that every third-party image, font, or reference can be shared.

See [docs/reuse-checklist.md](docs/reuse-checklist.md) for a more detailed release checklist.
