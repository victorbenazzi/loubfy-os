# Memory

This folder stores persistent project memory for agents.

Agents should read `memory/project-state.yml` and `memory/persistent-memory.md` before strategic work.

## Files

- `project-state.yml`: machine-readable setup state and key variables.
- `persistent-memory.md`: durable facts, preferences, and recurring context.
- `decisions.md`: important decisions and why they were made.
- `open-questions.md`: unknowns to resolve later.
- `competitors.md`: competitor and alternative notes.
- `content-pillars.md`: durable content pillar decisions.

Do not use memory for drafts. Drafts belong in `content/`.
